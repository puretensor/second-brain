## Session: Phase 3 Build
**Context:** pureMind Phase 3 -- Memory Search & Hybrid RAG

### Work Done
- Created puremind_chunks table in vantage DB (pgvector 0.5.1, fox-n1:30433). Schema: id, file_path, heading_path, chunk_index, content, embedding vector(768), content_tsv tsvector, file_hash. 4 indexes (HNSW, GIN, file, hash).
- Created tools/chunker.py: heading-aware markdown chunker. Splits on #{1-4}, tracks breadcrumb paths, fixed-window fallback for >2048 char sections with 20% overlap.
- Created tools/embed.py: nomic-embed-text-v1.5 (768-dim) via sentence-transformers. Supports search_document and search_query prefixes.
- Created tools/index.py: full + incremental indexing with SHA-256 change detection. Indexed 17 files -> 106 chunks in 7.2s. Incremental mode skips unchanged files.
- Created tools/search.py: hybrid BM25+vector search with RRF fusion (k=60). CLI interface with --limit, --json, --file-filter options.
- Created .claude/skills/puremind-search.md: Claude Code skill for on-demand vault search.
- Created tensor-scripts/hooks/puremind_index.sh: PostToolUse auto-index hook for vault changes.
- Enhanced daily_reflect.py: RAG context injection before Claude reflection prompt. Extracts topics from daily log, searches for historical context.
- Updated CLAUDE.md, README.md, project README with Phase 3 documentation.

### Decisions
- Used nomic-embed-text-v1.5 (768-dim) via sentence-transformers instead of Ollama (down) or all-MiniLM-L6-v2 (384-dim). Matches Nexus dimension for future cross-search.
- New table in existing vantage DB (not separate DB). Same infra as Alexandria/Nexus, separate table.
- Synchronous psycopg2 for CLI tools (simpler than async for Bash calls).
- Excluded soul.md and user.md from indexing (loaded at session start, don't need RAG).
- Ray embedding deferred: Python version mismatch (TC 3.12 vs fox 3.13) blocks ray.init(). Local embedding <2s for 106 chunks.
- Installed psycopg2-binary in user space (--break-system-packages on Ubuntu 24.04).
